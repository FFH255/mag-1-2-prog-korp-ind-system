# Задание 4. Перечень артефактов

## 4. Перечень артефактов

### 4.1. Правила именования артефактов

Имя файла артефакта формируется по шаблону:

```
[КОД_СТАДИИ]-[КОД_ЭТАПА]-[ТИП_ДОКУМЕНТА]-[ВЕРСИЯ].[РАСШИРЕНИЕ]
```

**Примеры:**

| Артефакт | Имя файла |
|---|---|
| Отчёт об обследовании, версия 1.0 | `S1-1-OBSLEDOVANIE-v1.0.pdf` |
| Техническое задание, версия 2.1 | `S3-2-TZ-v2.1.docx` |
| Логическая модель данных, версия 1.0 | `S5-2-ER-LOGICAL-v1.0.drawio` |

**Правила именования:**

- Допускаются только латинские буквы, цифры и символы `-`, `_`
- Версия обязательна в имени файла
- Пробелы в именах файлов запрещены

---

### 4.2. Правила версионирования

| Тип изменения | Версия | Пример |
|---|---|---|
| Первичное создание | v0.1 | Черновик |
| Доработка по замечаниям | v0.x | v0.2, v0.3 |
| Согласованная версия | v1.0 | Утверждено |
| Изменение после согласования | v1.x | v1.1, v1.2 |
| Принципиальный пересмотр | vX.0 | v2.0 |

- Версии до v1.0 — черновики, не являются базовой линией
- v1.0 и выше — официальные версии, изменения только через процесс управления изменениями
- История версий ведётся в листе согласования документа или в системе контроля версий

| Артефакт | Формат | Структура | Минимальный объём |
|---|---|---|---|

**Обязательные разделы ТЗ по ГОСТ 34.602-89:**

1. Общие сведения
2. Назначение и цели создания системы
3. Характеристика объектов автоматизации
4. Требования к системе (функциональные, нефункциональные, к надёжности, к безопасности)
5. Состав и содержание работ по созданию системы
6. Порядок контроля и приёмки системы
7. Требования к составу и содержанию работ по подготовке объекта автоматизации
8. Требования к документированию
9. Источники разработки
